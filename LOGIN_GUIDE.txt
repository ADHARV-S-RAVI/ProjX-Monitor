COMPLETE LOGIN FLOW:
==================================================

1. USER STARTS AT: MainPage.html (or index.html)
   - Displays landing page with features
   - Has "Log In" and "Sign Up" buttons
   - Shows demo login option in modal

2. USER CLICKS "Log In" → Opens Auth Modal
   - Option 1: Email/Password (Firebase)
   - Option 2: Google Sign-in (Firebase)
   - Option 3: Demo Login (SRN-based) ← NEW

3. DEMO LOGIN FLOW:
   - User enters valid Student SRN
   - Available SRNs: 24SUUBECS0047, 24SUUBECS0045, 24SUUBELCS001, etc.
   - User data stored in sessionStorage
   - Modal shows ROLE SELECTION

4. ROLE SELECTION:
   ├─ STUDENT ROLE
   │  └─ Redirects to: StudPage.html
   │     - Dashboard with projects
   │     - Team formation
   │     - Marks tracking
   │     - AI mentor assistant
   │
   └─ TEACHER ROLE
      └─ Redirects to: TeacherPage.html
         - Class & subject selection
         - Marks management
         - Team evaluation
         - Student roster

5. SESSION MANAGEMENT:
   - User stored in sessionStorage (survives page refresh within session)
   - Each page validates session on load
   - If no session → redirect back to MainPage.html
   - Logout button clears session & returns to MainPage
